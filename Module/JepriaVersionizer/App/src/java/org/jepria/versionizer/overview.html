Описание

Запуск производится нативным jdk из командной строки.
При этом необходимо указать опцию кодировки UTF-8, а в класспасе указать пути ко всем зависимостям (в частности, jsoup), а в конце — Main-класс.

Модули, которые участвуют в выпускаемой версии, можно конфигурировать через агрументы командной строки и через UI. Конфигурация модулей через аргументы командной строки:
  • --core C:\work\Jepria 1.0.0 — конфигурация Jepria с обязательным указанием пути до папки с модулем и имени версии
  • --toolkit C:\work\JepriaToolkit 2.0.0 — конфигурация JepriaToolkit с обязательным указанием пути до папки с модулем и имени версии
  • --application C:\work\JepriaShowcase 3.0.0 — конфигурация прикладного модуля с обязательным указанием пути до папки с модулем и имени версии
Модули, конфигурация которых не указана в аргументах командной строки, конфигурируются через UI. Опция --no-ui-conf использует только конфигурацию модулей из агрументов командной строки.
Примеры:
«Голый» запуск (конфигурация всех модулей запрашивается через UI): 
java -Dfile.encoding=UTF-8 -cp "C:\work\Bin\build\org\jepria\versionizer\core\1.0.0\jepria-versionizer-core-1.0.0.jar;C:\work\Bin\build\org\jsoup\jsoup\1.11.2\jsoup-1.11.2.jar" org.jepria.versionizer.Main
Запуск с указанной конфигурацией Jepria (конфигурация toolkit и application будет запрашиваться через UI): 
java -Dfile.encoding=UTF-8 -cp "C:\work\Bin\build\org\jepria\versionizer\core\1.0.0\jepria-versionizer-core-1.0.0.jar;C:\work\Bin\build\org\jsoup\jsoup\1.11.2\jsoup-1.11.2.jar" org.jepria.versionizer.Main --core C:\work\Jepria 1.0.0
Запуск с указанной конфигурацией Jepria (конфигурация toolkit и application не будет запрашиваться): 
java -Dfile.encoding=UTF-8 -cp "C:\work\Bin\build\org\jepria\versionizer\core\1.0.0\jepria-versionizer-core-1.0.0.jar;C:\work\Bin\build\org\jsoup\jsoup\1.11.2\jsoup-1.11.2.jar" org.jepria.versionizer.Main --core C:\work\Jepria 1.0.0 --no-ui-conf
